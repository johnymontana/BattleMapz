Why this is a bad visual

This visual shows employment in book, periodical and music stores from 2001-2011. This visual is a bad example because it uses a non-standard method to display time-series data. With time-series data typically the x-axis is used to represent time. Instead this visual uses the x-axis to show employment levels. Time is represented only by a label on the scatter plots themselves. This results in initial confusion: at first glance, employment in book stores appears to be growing linearly. After orienting themselves to the visual, the viewer is able to discern that in fact employment in the industry is shrinking linearly.

I also find the use of the size of bubble to represent average employment per establishment troubling. In general this is a good idea as it allows the visual to convey another dimension of information, however the way this is implemented in this visual is less than ideal. Because average employment per establishment is essentially the x-axis value divided by the y-axis value and since the overall size of the industry has been changing linearly, the average employment per establishment has been relatively unchanged. Therefore, all bubbles appear to be the same size. This leaves the viewer confused. "Am I missing something?" the viewer wonders.

Why this is a good visual

I find this data visualization particularly appealing. It allows for the type of comparison across industry that is the focus of my hypothesis for this project. Consistent with Tufte's principle of maximizing the "data-ink ratio" there is quite a bit of information being displayed in this single graphic. Interactive features allow the viewer to select industries to view detailed information or zoom in on a section of the chart to compare industries. This level of interactivity allows the visualization to maximize data density, without losing legibility or information since the user can zoom in and/or select individual points for more information. Some visuals use interactive features as a gimmick, resulting in chartjunk or focusing on design rather than the actual display of data, however I think this visual provides the right level of interactivity without impacting graphical integrity. This graphic became an inspiration for me as I moved forward with the project and indeed Visual #4 is an attempt to reproduce this graphic and some of the interactive features, but with data pertinent to the investigation of our hypothesis.

About Visual #1

This visual shows average annual earnings in Missoula, MT for the food manufacturing and custom computer programming industries from 1990-2012. To maintain graphical integrity and avoid distortions all values are adjusted for inflation by converting to 2012 equivalent dollars using the Consumer Price Index.

This graph shows data consistent with our hypothesis: it is clear that average annual earnings in the computer programming industry have been growing steadily while food manufacturing salaries have not even been keeping up with inflation. It is interesting to note that as recently as 2000 food manufacturing on average paid more than computer programming, but since then salaries have diverged dramatically.

About Visual #2

Number of establishments per industry is used as a measurement of industry size in this graphic. I debated plotting a growth rate, rather than the actual number of establishments, however I felt that since the initial size of the industries were similar any conversion to a growth rate would not add any benefit and might further distort interpretation of the data.

We can see that initally food manufacturing is a larger industry, however computer programming quickly grows to dwarf the food manufacturing industry. This is consistent with our hypotheis: it is better to be employed in an industry that is consistently growing, rather than one that is stagnant.

It is also important to note that number of establishments might not be a perfect measurement of the size of an industry. Employment level, the number of persons employed per industry, is also available and is indeed used in Visual #4.

	  var addRegression = function(series) {
	    var num_series = series.length;
	    for (var i = 0; i<num_series; i++) {
	      var obj = series[i];
	      var x = [];   
	      var y = [];
	      var s_x = 0;
	      var s_y = 0;
	      var s_xx = 0;
	      var s_yy = 0;
	      var s_xy = 0;
	      for (var j = 0; j<obj.data.length; j++) {
		x.push(obj.data[j].x);
		y.push(obj.data[j].y);
		s_x += obj.data[j].x;
		s_y += obj.data[j].y;
		s_xy += obj.data[j].x * obj.data[j].y;
		s_xx += obj.data[j].x * obj.data[j].x;
		s_yy += obj.data[j].y * obj.data[j].y;
	      }

	      var m = (x.length * s_xy - s_x * s_y) / (x.length * s_xx - (s_x*s_x));
	      var b = (1 / x.length) * s_y - m * (1/ x.length) * s_x;

	      var obj2 = {};
	      obj2.name = obj.name + " regression";
	      obj2.color = obj.color;
	      obj2.data = [];
	      obj2.renderer = 'line';
	      for (var k=0; k < x.length; k++) {
		obj2.data.push({x: x[k], y: (m*x[k] + b)}); 
	      }
    
	      series.push(obj2);
	    }
	  return series;
	}
